Changelog

TAssetBundle의 모든 주요 변경 사항을 이 파일에 기록합니다.

5.0.0

  • ⚠️ Breaking (최소 Unity 버전): Unity 2021.3 LTS로 상향 (기존 2019.4).
  • ⚠️ Breaking (폴더 구조): Extensions가 Assets/TAssetBundleExtensions/로 이동했습니다. 업그레이드 후 기존 Assets/TAssetBundle/Scripts/Extensions/ 폴더를 삭제하세요. using TAssetBundle.Extensions; 네임스페이스는 그대로 유지됩니다. 에디터 감지기가 기존 폴더가 남아 있으면 경고를 표시합니다.
  • 🆕 Lite 에디션 추가 — Lite(무료)는 코어 엔진과 플러그인을 제공하고, Pro는 productivity 레이어(AssetManager, LinkAsset, PrefabPoolManager, PathAliasTable, Asset Usage Tracker)를 추가합니다. 패키지 내 lite_vs_pro_ko.md 참조.

4.7.0

  • [AssetManager] Path Aliases 추가 — 접두어를 폴더에 매핑하며, 다중 폴더 우선순위 검색으로 DLC 오버라이드를 지원합니다. PathAliasTable 또는 PrefixAliasResolver로 설정합니다.
  • 실패한 에셋 로드는 캐시에서 자동 정리되어, 재시도 시 더 이상 UnloadUnusedAssets를 호출할 필요가 없습니다
  • 동일 경로를 다른 타입으로 로드하면 InvalidCastException 대신 명확한 에러를 반환합니다
  • 다중 코루틴 초기화와 풀 교체 중 발생하던 prefab pool race를 강화했습니다

4.6.1

  • [Play Asset Delivery] Pack 번들(install-time, fast-follow, on-demand)은 패킹 전 LZMA에서 LZ4로 재압축됩니다 — 모든 pack 타입은 로딩 전에 디바이스에 안착하므로, LZMA의 작은 크기보다 LZ4의 빠른 압축 해제 속도가 런타임 성능에 유리합니다.
  • Manifest Inspector 플러그인이 다중 편집을 지원합니다 — Project 창에서 여러 manifest를 선택하면 플러그인 설정(예: PAD 전송 타입)을 한 번에 모두 적용합니다. 플러그인 작성자에게 breaking: IManifestEditorPluginModule.OnManifestInspectorGUI가 단일 manifest 대신 TAssetBundleManifest[]를 받습니다.

4.6.0

  • [Android] Google Play Asset Delivery 플러그인 추가 — install-time / fast-follow / on-demand 전송 방식, 자동 AAB 통합, OnDownloadProgress 이벤트, 동시 다운로드 중복 제거, Split Application Binary 호환성 지원
  • UpdateTempCatalogAsync 추가 — 활성 카탈로그를 교체하지 않고 카탈로그 업데이트 여부와 다운로드 크기를 확인합니다
  • TagUtil.SetDirty가 dirty 표시 전 IsPersistent를 확인하도록 수정
  • CompositionStrategy Data 클래스에 [Serializable] 추가, 불필요한 SetDirty 호출 제거

4.5.0

  • Cross-Reference Analyzer 추가 — 빌드 시점에 번들 간 에셋 참조를 감지하고 시각화
  • Dependency Checker를 TreeView + SearchField 기반으로 재설계
  • Browser의 manifest 헤더에서 Builtin/Encrypt/Tags를 한눈에 표시
  • ManifestPluginData 시스템 추가 — manifest별 플러그인 데이터 + 빌드 훅
  • 플러그인 Inspector UI 개선
  • 안전한 예외 처리를 위해 async void를 async Task로 전환
  • UnityWebRequestError에 enum 기반 errorType 추가
  • 카탈로그/AssetBundle 데이터 손상 복원력 및 IO 에러 보호 강화
  • AssetBundleRuntimeInfo.TryGetCustomData() 추가
  • "Shrink All"을 "Collapse All"로 명칭 변경

4.4.0

  • TAssetBundleManifest에 에셋 검색 API 추가 (SearchAssets, TryGetAssetInfo)
  • TAssetBundle Browser에 에셋 검색 기능 추가

4.3.1

  • GetNotIncludedAssets()이 자식 manifest를 포함한 manifest 트리 전체에 걸쳐 에셋 포함 여부를 정확히 평가하도록 수정
  • 부모 폴더나 이미 포함된 에셋이 미포함으로 잘못 보고되던 케이스를 해결
  • 내부 에셋 포함 검사 및 관련 로직 개선

4.3.0

  • 런타임 경로 기반 에셋 로딩을 위한 FilePathCollector 사용 예제 샘플 추가
  • 더 명확한 이해를 돕기 위해 샘플 코드에 추가 주석 보강
  • 다운로드 결과 처리에 상세 에러 정보 포함하도록 개선
  • 빌드 프로세스 최적화 및 안정성 향상을 위한 사소한 내부 리팩터링

4.2.0

  • 런타임에 에셋 경로 정보를 수집하는 FilePathCollector 추가
  • 확장 가능한 플러그인 시스템 추가
  • 카탈로그의 RemoteUrl이 캐싱되어, 새 RemoteUrl 적용 전까지 이전에 다운로드된 에셋에 접근할 수 없던 문제 수정
  • 그 외 최적화 및 사소한 버그 수정

4.1.1

  • DefaultAssetCatalogComparer 비교 연산 버그 수정

4.1.0

  • 빌드 시간을 크게 단축하는 빌드 프로세스 최적화

4.0.0

  • API를 명확히 하기 위해 TAssetBundle.AssetManager를 TAssetBundle.TAssetBundles로 변경
  • TAssetBundle 기반으로 손쉬운 에셋 로드/언로드, prefab pooling, 객체 사용 추적을 제공하는 확장 AssetManager 추가
  • 에디터에 Asset Usage Tracker 추가 — 플레이 모드에서 특정 에셋을 사용 중인 객체를 정규식으로 보고/정렬/검색 가능
  • 자주 사용하는 prefab의 효율적인 풀링을 위한 새로운 PrefabPoolManager 추가
  • 비디오 에셋이 포함된 에셋 번들을 비압축으로 처리
  • Asset Reference Tracker에 정렬 및 정규식 검색 기능 추가
  • 다양한 버그 수정, 리팩터링 및 최적화

3.11.1

  • Unity 2022 이후 버전에서 TAssetBundleManifest 생성 시 list 필드의 null reference 에러 수정

3.11.0

  • 에셋 번들 빌드 시 builtin 에셋 번들을 별도 폴더로 출력하는 옵션 추가. 옵션 이름은 "Builtin Asset Bundles Use Separate Folder"
  • LZMA로 압축된(작은 크기) builtin 에셋 번들이 처음 사용 시 LZ4로 재압축되도록 변경
  • WebGL 플랫폼에서는 native 캐싱이 동작하지 않으므로 Unity의 기본 AssetBundle provider를 사용하지 않도록 수정. 대신 전용 managed AssetBundle provider를 사용. Unity의 기본 AssetBundle provider 사용 시 다운로드 크기를 알 수 없던 이전 버전의 버그를 수정합니다.

3.10.0
